# ShemOS
**A Human–AI Collaboration Framework**
*Version 1.1 | By Jill Shem | [jillshem.com](https://jillshem.com)*

---

ShemOS is a portable framework for working with AI as a genuine thinking partner. It was built under real conditions, pressure-tested in live sessions, and documented as it stabilised.

It is designed for humans whose cognition doesn't fit the default model — and for anyone building tools that need to hold more than one kind of human at once.

---

## What's in this repo

- `ShemOS.md` — the full public framework, including both layers, the cognitive model, 14 documented AI failure modes, operating principles, and license
- `ShemOS-commands.md` — the shorthand command vocabulary: glitter, glimmer, glow, and glint. Prompt templates, a publishing pipeline, and instructions for building your own commands

---

## What use looks like

**Permitted:**
- An AI safety researcher reads ShemOS and uses it to inform a paper on human-AI interaction
- An AI product team implements the two-layer protocol to improve how their tool handles cognitively dense users
- A developer runs ShemOS through Claude to build a personal collaboration workflow
- A university course uses it as a framework for discussing AI design failures

**Not permitted:**
- An AI company ingests this text into a model training pipeline
- A platform uses ShemOS as labelled data to fine-tune a commercial model
- A company white-labels the framework in a product without a written agreement

The line is between *using the framework* and *absorbing the text into a model*. The first is the point. The second requires a conversation.
